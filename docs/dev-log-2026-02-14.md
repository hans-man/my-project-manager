# my-project-manager 개발 로그 (2026-02-14)

## 1. 프로젝트 목표

별도의 DB 설치 없이, 문서 양식에 따라 데이터 구조가 동적으로 변경되는 프로젝트 관리 솔루션 구축.

**핵심 요구사항:**
- 진척 관리
- 프로그램 본수 관리
- 인력 관리
- 산출물 관리
- 개발 소스 점검

## 2. 아키텍처 및 기술 스택 결정

- **핵심 아키텍처:** 파일 기반 시스템. Markdown 파일의 **Frontmatter**를 활용하여 동적 데이터 구조 구현.
- **데이터 저장:** `data` 폴더 내에 Markdown 파일(.md)로 프로젝트 데이터(인력, 작업 등)를 저장.
- **언어/환경:** Node.js
- **UI:** 초기에는 CLI(명령어 기반 인터페이스)로 개발, 향후 Electron/Tauri 등으로 GUI 확장 가능.
- **버전 관리:** Git 및 GitHub

## 3. 초기 환경 설정 (Initial Setup)

### 3.1. GitHub 리포지토리 생성 및 로컬 복제

- GitHub에 `my-project-manager` 비공개(Private) 리포지토리 생성.
- 로컬 PC에 Git 설치 및 사용자 정보(`user.name`, `user.email`) 설정.
- `git clone` 명령으로 로컬에 리포지토리 복제.

### 3.2. 프로젝트 폴더 구조 생성

- ` 관심사의 분리` 원칙에 따라 폴더 구조 설계.
- 생성된 폴더:
  - `data/`
  - `docs/`
  - `src/cli/commands/`
  - `src/core/`
  - `src/utils/`
  - `tests/`

### 3.3. Node.js 프로젝트 초기화 및 `.gitignore` 생성

- `npm init -y` 명령을 실행하여 `package.json` 생성.
- `node_modules`, `.env` 등 불필요한 파일/폴더를 Git 추적에서 제외하기 위해 `.gitignore` 파일 생성.

## 4. 주요 실행 명령어 요약

```sh
# Git 리포지토리 복제
git clone https://github.com/hans-man/my-project-manager.git C:\Users\angel\my-project-manager

# 폴더 생성
mkdir data
mkdir src
mkdir tests
mkdir docs
mkdir src/cli
mkdir src/core
mkdir src/utils
mkdir src/cli/commands

# Node.js 프로젝트 초기화
npm init -y

# Git 사용자 정보 설정
git config --global user.email "your_email@example.com"
git config --global user.name "Your Name"

# Git 변경사항 커밋 및 푸시
git add .
git commit -m "feat: Initial project structure and npm init"
git push origin main
```

## 5. 인력 관리 기능 구현 (User Management)

### 5.1. 라이브러리 설치
- `gray-matter`: Markdown Frontmatter 파싱을 위해 설치.
- `commander`: 강력한 CLI 명령어 인터페이스 구축을 위해 설치.
```sh
npm install gray-matter commander
```

### 5.2. `user add` 기능 구현
- `data/people` 폴더 생성: 사용자 데이터 파일 저장 위치.
- `src/core/userService.js` 생성: `addUser` 함수를 구현하여, `gray-matter`를 통해 사용자 정보를 Frontmatter로 갖는 Markdown 파일을 생성하는 핵심 로직 담당.
- `src/cli/commands/user.js` 생성: `commander`를 사용하여 `user add <name>` 명령어를 정의하고 `userService.addUser` 함수를 호출.
- `src/cli/main.js` 생성: `commander` 프로그램을 초기화하고 모든 CLI 명령어를 등록하는 진입점 역할.
- **테스트:** `node src/cli/main.js user add "Hans Man" --email "hans@example.com"` 명령으로 `data/people/Hans-Man.md` 파일 생성 확인.

### 5.3. `user list` 기능 구현
- `src/core/userService.js` 수정: `listUsers` 함수를 추가하여 `data/people` 디렉토리의 모든 사용자 파일을 읽고 파싱하여 배열로 반환.
- `src/cli/commands/user.js` 수정: `user list` 명령어를 추가하고 `userService.listUsers`를 호출하여 결과를 `console.table`로 출력.
- **테스트:** `node src/cli/main.js user list` 명령으로 사용자 목록이 테이블 형태로 출력됨을 확인.
