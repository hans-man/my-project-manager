# my-project-manager 개발 로그 (2026-02-14)

## 1. 프로젝트 목표

별도의 DB 설치 없이, 문서 양식에 따라 데이터 구조가 동적으로 변경되는 프로젝트 관리 솔루션 구축.

**핵심 요구사항:**
- 진척 관리
- 프로그램 본수 관리
- 인력 관리
- 산출물 관리
- 개발 소스 점검

## 2. 아키텍처 결정: 웹 기반 클라이언트-서버 모델로 전환

### 2.1. 기존 CLI + 파일 기반 아키텍처의 한계 인식
- 초기 제안된 CLI + 로컬 파일 기반 방식은 단일 사용자 환경에서는 적합하나, 다중 사용자 동시 접속 및 데이터 동기화 요구사항을 충족시키기 어려움.

### 2.2. 클라이언트-서버 아키텍처 도입 결정
- 여러 사용자의 동시 접근 및 데이터 동기화를 위해 중앙 집중식 데이터베이스 및 API 서버 도입 결정.
- "외부망과 연결되지 않은 환경" 제약 조건에 따라 내부 네트워크에 서버 호스팅.
- "동적 데이터 구조" 요구사항을 만족하기 위해 NoSQL 문서 데이터베이스 채택.

### 2.3. 웹 기반으로 클라이언트 변경 결정
- 배포 용이성, 사용자 경험(UI/UX), 크로스 플랫폼 지원을 위해 CLI 클라이언트 대신 웹 기반 클라이언트를 선택.
- 각 사용자는 웹 브라우저를 통해 내부 네트워크에 호스팅된 웹 애플리케이션에 접속.

## 3. 기술 스택 결정

- **프론트엔드 (클라이언트):** React (create-react-app 사용)
- **백엔드 (API 서버):** Node.js (Express)
- **데이터베이스:** MongoDB (NoSQL 문서 데이터베이스)
- **버전 관리:** Git 및 GitHub

## 4. 초기 환경 설정 및 백엔드 구성 (Initial Setup & Backend)

### 4.1. GitHub 리포지토리 설정 및 구조 변경

- GitHub에 `my-project-manager` 비공개(Private) 리포지토리 유지.
- 기존 CLI 관련 `src` 및 `data` 디렉토리 삭제.
- 새로운 모노레포(Monorepo) 구조 채택: `backend/`, `frontend/` 디렉토리 생성.
  - `data/people/Hans-Man.md` 파일은 백업 후 삭제.

### 4.2. 백엔드 프로젝트 초기화 및 의존성 설치

- `backend/` 디렉토리 내에서 `npm init -y`로 Node.js 프로젝트 초기화.
- 필요한 패키지 설치: `express`, `mongoose`, `cors`, `dotenv`.

### 4.3. 백엔드 서버 기본 설정 (`backend/server.js`)

- Express 애플리케이션 초기화 및 미들웨어 설정 (CORS, JSON 파싱).
- `dotenv`를 사용하여 `.env` 파일에서 환경 변수 로드.
- MongoDB 연결 설정: `mongoose`를 사용하여 `MONGO_URI` 환경 변수 값으로 데이터베이스 연결 시도.
- 기본 라우트 (`/`) 설정: 서버 작동 여부 확인용 "Hello World" 응답.
- 서버 포트 설정 (`PORT` 환경 변수 사용).

### 4.4. 환경 변수 파일 생성 (`backend/.env`)

- 백엔드 연결 정보(예: `PORT`, `MONGO_URI`)를 위한 `.env` 파일 생성 및 플레이스홀더 값 지정.

### 4.5. `.gitignore` 업데이트

- `backend/node_modules/` 및 `backend/.env`를 `.gitignore`에 추가하여 Git 추적에서 제외.

## 5. 프론트엔드 구성 (Frontend Setup)

### 5.1. React 프로젝트 초기화
- `frontend/` 디렉토리 내에서 `npx create-react-app .` 명령으로 React 프로젝트 초기화.
- 유지 보수 용이성을 위해 `create-react-app` 선택.

### 5.2. 의존성 설치
- 백엔드 API 요청을 위한 `axios` 라이브러리 설치.

### 5.3. 백엔드 통신 설정 (`frontend/src/App.js`)
- `frontend/src/App.js` 파일을 수정하여 컴포넌트 마운트 시 백엔드의 기본 라우트(`http://localhost:5000` 또는 프로덕션 환경에서는 동일 오리진)에서 메시지를 가져와 화면에 표시하도록 구현.

### 5.4. `.gitignore` 업데이트
- `frontend/node_modules/` 및 `frontend/.env`를 루트 `.gitignore`에 추가하여 Git 추적에서 제외.

## 6. 주요 실행 명령어 요약 (최신화)

```sh
# Git 리포지토리 복제 (최초 1회)
git clone https://github.com/hans-man/my-project-manager.git C:\Users\angel\my-project-manager

# 기존 CLI/파일 기반 관련 파일 제거 및 백엔드/프론트엔드 폴더 생성
# (data/people/Hans-Man.md는 백업 후 삭제)
Remove-Item -Recurse -Force data
Remove-Item -Recurse -Force src
mkdir backend
mkdir frontend

# 백엔드 프로젝트 초기화 및 의존성 설치
cd backend
npm init -y
npm install express mongoose cors dotenv
cd .. # 다시 프로젝트 루트로 이동

# .gitignore 업데이트
# (backend/node_modules/, backend/.env, frontend/node_modules/, frontend/.env 추가)

# 프론트엔드 프로젝트 초기화 및 의존성 설치
cd frontend
# npx create-react-app . --use-npm # --use-npm 플래그를 사용하여 npm 사용 명시 (자동으로 npm 사용)
& "C:\Program Files\nodejs\npx.cmd" --yes create-react-app .
& "C:\Program Files\nodejs\npm.cmd" install axios
cd .. # 다시 프로젝트 루트로 이동

# Git 변경사항 커밋 및 푸시
# (이전 커밋에 백엔드 설정 포함)
git add .
git commit -m "feat: Frontend setup (React with create-react-app)"
git push origin main
```
